6. Decorder(디코더)
	-  Encoder(인코더)와 정반대의 역할을 하는 조합 회로로 n개의 입력라인을 2^n개의 출력 라인으로 변환해서 입력의 이진 코드를 해당하는 단일 출력 라인의 신호로 변환하는 역할을 한다. 
		- ex) 3X8 디코더에서 입력라인은 A0,A1,A2 총 3개의 입력이 0과 1 이진코드로 입력되면 입력에 따라 출력라인인 D0~7 까지중 한가지 단일 출력라인에 신호를 준다.
7. Shifter(시프트)
	- 연산자중 하나로 비트를 이동시켜 2의 제곱으로 곱하거나 나누는 효과를 내는 연산자다. 왼쪽 오른쪽 모두 이동 가능하며 생긴 모양은 '원하는 숫자 (원하는 방향 <<(왼쪽) 혹은 >>(오른쪽)) 이동할 칸' 이다.
		- ex) 10 << 2 라면 원래 2진수로 표현한 10은 '0000 1010'으로 표현 되는데 왼쪽으로 2칸을 이동하니 '0010 1000'으로 40이 된다.
8. Adress Bus(주소 버스)
	- 컴퓨터 내부에서 데이터, 주소, 제어 신호들을 전달하는 공유 통신 경로인 Bus(버스) 중 하나로 데이터를 주고 받는 주소를 지정하는 통로다. 어디에서 읽는지, 어디에 사용하는지 등 주소 정보들을 전달하는데 다른 두 종류의 버스완 다르게 CPU에서 메모리로 전달 혹은 I/O 장치로만 전달한다. 즉 데이터의 흐름이 단방향이다.					
9. Data Bus(데이터 버스)
	- Bus(버스) 중 하나로 CPU와 메모리, I/O 장치 사이에서 데이터 자체를 주고 받는 역할을 한다. 그렇기 때문에 데이터의 흐름이 양방향으로 CPU가 데이터를 읽고 쓰기도 한다. 또한 예시로 64비트 데이터 박스는 한번에 64비트의 데이터를 전송하는것이 가능하다.
10.	Control Bus(컨트롤 버스)
	- Bus(버스) 중 하나로 Read(읽기), Write(쓰기), Interrupt(인터럽트), Clock(클럭)등 제어 신호들을 전달하는 역할을 한다. 그렇기 때문에 데이터의 흐름이	양방향으로 CPU와 주변장치 간의 명령과 상태 신호를 교환한다.		
11. 명령어 사이클
	- CPU가 메모리에서 하나의 기계어 명령어를 가져와 해석하고 실행하는 일련의 기본 작업 주기로 크게 총 4단계로 이뤄져 이 과정을 컴퓨터가 켜질때부터 꺼질때까지 반복한다.
	- 첫번째 단계는 인출 사이클(fectch cycle)로 CPU가 다음 실행할 명령어를 메모리에서 CPU 내부로 가져오는 작업이다.
	- 두번째 단계는 간접 사이클(Indirect cycle)로 명령어에 사용될 데이터가 메모리에 저장되어 있을 때 메모리에 있는 실제주소를 가져와 CPU에 적재시키는 작업으로 만약 데이터에 바로 접근하는 명령어라면 이 단계는 생략된다.
	- 세번째 단계는 실행 사이클(Execution cycle)로	 가지고 온 명령어를 해석(Decode)하고, 그에 해당하는 연산을 실제 실행하는 작업이다.
	- 네번째 단계는 인터럽트 사이클(Interrupt cycle)로 프로그램 실행 중 예기치 않은 상황이 발생하여 처리가 필요할 때 수행되는 작업으로 현재 실행중인 명령어의 상태를 안전하게 저장하고 인터럽트 처리루틴으로 분기하고, 처리가 끝나면 다시 원래의 사이클로 돌아온다.
